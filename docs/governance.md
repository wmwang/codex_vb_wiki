# 內容治理與審核流程

## 角色
- **Owner**：功能模組負責人，負責內容正確性。
- **Reviewer**：具備審核權限，確保格式一致與安全性。
- **Editor**：新增/更新內容，需經 Reviewer 核准。

## 流程
1. 自動抽取並生成初稿。
2. Editor 補齊細節（流程、錯誤處理、注意事項）。
3. Reviewer 審核：正確性、敏感資訊遮罩、格式一致性。
4. Owner 確認並合併。
5. 變更記錄與稽核紀錄自動保存。

## 規範
- 重要模組的變更需對應到 PR/變更單。
- 每次更新需標註影響範圍與相依模組。
- 敏感資訊不得直接寫入，需使用遮罩或參考內部系統。
